# AI用例智能体三种模式功能说明

## 功能概述

为AI用例智能体新增了三种生成模式，用户可以根据需求复杂度选择合适的模式，以获得不同覆盖度和数量的测试用例。

## 三种模式详细说明

### 1. 全覆盖模式（comprehensive）- 复杂需求
- **适用场景**: 复杂的业务需求，需要全面的测试覆盖
- **用例数量**: 每个功能点生成 5-15 个用例，复杂功能可达 20+ 条
- **覆盖维度**: 
  - 正向流程：主流程 + 各种正常分支场景
  - 异常处理：各类错误输入、异常状态、系统异常
  - 边界值：上下边界、临界值、极限值
  - 业务规则：权限控制、状态流转、数据校验、业务约束
  - 兼容性：不同角色、不同状态、不同环境
  - 性能与安全：大数据量、并发、敏感操作
- **设计方法**: 等价类划分、边界值分析、场景法、状态转换、组合测试等

### 2. 平衡模式（balanced）- 中等需求
- **适用场景**: 中等复杂度的需求，需要平衡覆盖度和效率
- **用例数量**: 每个功能点生成 3-8 个用例
- **覆盖维度**:
  - 正向流程：主流程 + 重要分支场景
  - 异常处理：主要错误输入、关键异常状态
  - 边界值：关键边界点（最小值、最大值、0值）
  - 业务规则：核心权限控制、主要状态流转、重要数据校验
- **设计原则**: 重点覆盖核心功能和高频使用场景，效率优先

### 3. 质量模式（quality）- 简单需求
- **适用场景**: 简单需求或快速迭代场景，追求精准高效
- **用例数量**: 每个功能点生成 2-5 个用例
- **覆盖维度**:
  - 正向流程：核心主流程验证
  - 异常处理：最重要的错误场景
  - 边界值：关键边界点验证
  - 业务规则：核心业务逻辑验证
- **设计原则**: 精准高效，每个用例都有明确价值，质量至上

## 前端交互设计

在"智能生成用例"按钮上方新增模式选择区域：

```
生成模式
○ 全覆盖模式（复杂需求）
○ 平衡模式（中等需求）
○ 质量模式（简单需求）

[智能生成用例]
```

- 默认选择"全覆盖模式"
- 用户可以根据需求复杂度选择合适的模式
- 模式选择会影响AI生成的用例数量和覆盖度

## 技术实现

### 后端修改

1. **提示词扩展** (`backend/apps/ai_testcase/services/prompts.py`)
   - 新增三种模式的专用提示词
   - 支持纯文本和多模态两种场景
   - 提供模式选择函数

2. **API客户端更新** (`backend/apps/ai_testcase/services/kimi_client.py`)
   - 流式和非流式生成方法都支持模式参数
   - 多模态生成方法支持模式参数

3. **API接口更新** (`backend/apps/ai_testcase/views.py`)
   - 流式生成接口支持mode参数
   - ViewSet接口支持mode参数

4. **序列化器更新** (`backend/apps/ai_testcase/serializers.py`)
   - GenerateRequestSerializer新增mode字段
   - 支持三种模式的选择验证

### 前端修改

1. **组件更新** (`frontend/src/pages/workmind/ai_testcase/AiTestcaseGenerator.vue`)
   - 新增模式选择单选框组
   - 新增generationMode状态变量
   - 生成请求时传递模式参数
   - 新增相关样式

## 向后兼容性

- 所有现有API保持兼容，mode参数为可选，默认值为'comprehensive'
- 现有的提示词保持不变，作为全覆盖模式使用
- 前端默认选择全覆盖模式，保持原有行为

## 使用建议

- **复杂业务系统**: 使用全覆盖模式，确保全面的测试覆盖
- **一般功能开发**: 使用平衡模式，在覆盖度和效率间取得平衡
- **快速原型验证**: 使用质量模式，快速获得核心测试用例
- **敏捷开发迭代**: 根据迭代阶段选择合适模式，早期可用质量模式，后期使用全覆盖模式
