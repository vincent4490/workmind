# AI 用例智能体生成模式说明（两种）

## 功能概述

用户可根据需求选择生成模式，在**全覆盖**与**聚焦（功能与业务）**之间切换，以控制用例条数与覆盖深度。

## 模式说明

### 1. 全覆盖模式（`comprehensive`）

- **适用场景**：发布前、强合规、复杂业务或需穷尽类测试设计方法时
- **用例数量**：每功能点约 5–15 条，复杂功能可达 20+ 条
- **覆盖维度**：正向、异常、边界、业务规则、兼容性、性能与安全等尽量铺全；等价类、边界、场景、状态、组合等方法学写全

### 2. 聚焦模式（`focused`）

- **适用场景**：迭代交付、冒烟、核心回归；优先验证主流程与核心业务规则
- **用例数量**：每功能点约 3–10 条，复杂功能一般不超过约 15 条
- **覆盖维度**：主流程与关键分支、主要异常与关键边界、核心业务规则；不追求等价类穷尽与兼容性/性能/安全专项（除非需求明确写出）

## 前端交互

在「智能生成用例」上方选择：

- 全覆盖模式（高覆盖、方法学与异常场景尽量铺全）
- 聚焦模式（功能与业务优先，条数精简）

默认：**全覆盖**。请求参数 `mode` 为 `comprehensive` 或 `focused`。

## 技术实现要点

- 提示词：`backend/apps/ai_testcase/services/prompts.py`（纯文本与多模态各两套 system prompt）
- 序列化：`GenerateRequestSerializer.mode` 枚举上述两值
- Kimi 调用：`KimiClient` 将 `mode` 传入 `get_testcase_prompt` / `get_testcase_prompt_multimodal`

## 使用建议

- **全量回归 / 强质量要求**：`comprehensive`
- **快速验证核心路径与业务规则**：`focused`
