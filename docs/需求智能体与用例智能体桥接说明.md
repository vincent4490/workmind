# 需求智能体 → 用例智能体桥接说明

本文档说明从需求任务跳转到「AI 用例智能体」时的数据流与接口。**历史上曾支持「功能点 JSON 直传 + `generate-from-structure`」方案，已移除**；当前仅保留 **原文预填 + `generate-stream`**。

---

## 一、当前流程（唯一路径）

1. 用户在需求侧完成某次任务后，点击「生成测试用例」（或等价入口）。
2. 前端调用 `POST /api/ai_requirement/bridge-to-testcase/`，请求体：`{ "task_id": <id> }`。
3. 后端返回 JSON：
   - `status`: `"ok"`
   - `requirement_text`: 任务输入或结果摘要（供用例页预填）
   - `source_task_id`: 需求任务 ID（供前端展示或后续扩展）
4. 前端将 `requirement_text` 写入 `sessionStorage['ai_req_bridge_text']`（及可选的 `ai_req_bridge_source`），跳转到用例智能体页面。
5. 用例页 `onMounted` 读取并填入「需求描述」，用户点击 **「智能生成用例」** → `POST /api/ai_testcase/generate-stream/`（SSE），由模型从自然语言生成完整 **模块 / 功能点 / 用例** 树。

实现位置：`backend/apps/ai_requirement/views.py` 中 `bridge_to_testcase_view`（**不返回 `structure`**）。

---

## 二、桥接接口

| 项目 | 说明 |
|------|------|
| **URL** | `POST /api/ai_requirement/bridge-to-testcase/` |
| **请求体** | `{ "task_id": 123 }` |
| **响应** | `application/json`，字段见上一节 |

若任务无可用的需求文本，返回 **400**。

---

## 三、用例侧生成接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/ai_testcase/generate-stream/` | POST | 需求描述（+ 可选附件 FormData），SSE 流式返回，生成完整用例树 |

---

## 四、`downstream_testcases` 说明

`AiRequirementTask.downstream_testcases` 仍可用于记录「从该需求任务产生的用例生成记录 ID」等血缘信息。**原「仅在 `generate-from-structure` 完成时回写」的逻辑已随该接口删除**；若产品需要自动回写，应在 `generate-stream` 成功路径或其它约定位置另行实现。

---

## 五、前端要点

- 用例页：`AiTestcaseGenerator.vue` 从 `sessionStorage` 读取 `ai_req_bridge_text` 预填需求框（见组件内注释）。
- 不再存在 `aiGenerateTestcaseFromStructureStream` 或对 `/api/ai_testcase/generate-from-structure/` 的调用。
